#  Smart City AI Core: Local LLM Engine

Данный репозиторий содержит ядро искусственного интеллекта для проекта **Smart City Dashboard**. 
В отличие от стандартных решений на базе публичных API (ChatGPT / Gemini), наш проект использует **собственную развернутую и оптимизированную ИИ-инфраструктуру** на базе технологии Edge Computing.

## Архитектурное решение и Инновационность
Система работает как «Помощник Мэра», анализируя агрегированные данные города (трафик, качество воздуха) и генерируя управленческие решения в строгом формате JSON.

**Ключевые преимущества нашей архитектуры:**
1. **100% Data Privacy (Суверенитет данных):** Критически важные данные городской инфраструктуры не покидают защищенный контур серверов.
2. **Zero-Cost Operation:** Отсутствие зависимости от платных подписок и лимитов (Rate Limits) облачных провайдеров.
3. **Гибридная отказоустойчивость:** Основные вычисления происходят на локальном сервере, облачный API интегрирован исключительно как запасной канал (Fallback) на случай аппаратных сбоев.

## Технологический стек
* **Core Model:** Qwen 2.5 (7 Billion Parameters)
* **Inference Engine:** Ollama (с кастомным `Modelfile`)
* **Tunneling:** Ngrok (для интеграции с frontend/backend компонентами)
* **Optimization:** CUDA GPU Acceleration (Оптимизация под VRAM < 8GB)

## Инженерия модели (Prompt & Weights)
Мы создали собственную сборку модели `mayor-ai`, используя Ollama Modelfile. 
* Параметр `temperature` жестко зафиксирован на **0.1** для исключения галлюцинаций.
* Промпты переведены в строгий системный режим (System Role) с зашитой «шпаргалкой» по ПДК (Предельно допустимым концентрациям) и баллам пробок.
* Реализована защита от токенизационных сбоев (переход на чистый русский язык при формировании JSON).

## Бенчмарки и Производительность
В репозитории представлен модуль `stress_test.py` для тестирования нагрузки.
* **Среднее время ответа (Inference Time):** 3-4 секунды.
* **Потребление VRAM (Keep-Alive State):** 7.1 GB (Hot Start обеспечен).
* **Формат:** 100% успешная валидация JSON во всех краевых сценариях (от идеальных условий до аномальных данных датчиков).
